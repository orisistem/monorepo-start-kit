# Documentation and Management (`docs/`)

This directory is the central hub for all non-code artifacts required to run the software development company smoothly.

## Structure

- **`01-requirements/`**: Business requirements, PRDs (Product Requirements Documents), user stories (use `PRD-template.md` as starting point for AI-assisted spec generation).
- **`02-planning/`**: Roadmaps, sprint plans, resource allocation.
- **`03-architecture/`**: C4 models, ER diagrams, ADRs (Architecture Decision Records), technical specifications (`tech-spec-template.md`).
- **`04-api/`**: OpenAPI/Swagger specs, Postman collections.
- **`05-manuals/`**: Runbooks, user guides, admin manuals.
- **`06-meetings/`**: Minutes from daily stand-ups, planning, and retrospectives.
- **`10-ai-workflow/`**: AI-assisted development workflow — guidelines, prompts, and best practices for using AI tools (OpenCode, Antigravity) throughout the project lifecycle.
